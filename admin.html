Como restaurar (guia rápido)

A restauração é feita pelo script scripts/restore.js no backend (Railway ou local com DATABASE_URL de produção). Por padrão o script roda em dry-run: apenas simula e mostra o que seria restaurado, sem gravar nada.

# 1. Simulação (dry-run, padrão) a partir do último backup do bucket
node scripts/restore.js --from-storage latest

# 2. Simulação a partir de um arquivo local (v1 ou v2, .json/.gz/.enc)
node scripts/restore.js --file backups/aero-backup-2026-07-02.json.gz

# 3. Restaurar SOMENTE tabelas vazias (modo seguro recomendado)
node scripts/restore.js --from-storage latest --mode empty-only

# 4. Restaurar sobrescrevendo por id (upsert — usar com cautela)
node scripts/restore.js --file backup.json --mode upsert

O script nunca apaga dados: ele só cria (empty-only) ou cria/atualiza (upsert). O passo a passo completo, incluindo o drill trimestral de restauração, está em docs/RESTORE-DRILL.md no repositório.
